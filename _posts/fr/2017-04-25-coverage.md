Mécanismes internes de la machine virtuelle Java basés sur les défis et l'expérience acquis lors de l'implémentation de l'un des outils de couverture de code les plus populaires pour Java – JaCoCo, qui est la base du plugin Eclipse EclEmma primé.

Dans cette présentation, nous plongerons dans les entrailles de la JVM (bytecode, restrictions, chargeurs de classes) en répondant aux questions suivantes :
* Comment, en développant JaCoCo, nous avons trouvé des bogues dans OpenJDK ? Vous pouvez aussi le faire !
* Comment JaCoCo parvient-il à fonctionner dans un environnement OSGi ?
* Qu'est-ce qui le rend le plus rapide ?
* Et quels changements ont été nécessaires pour supporter Java 8 ? Java 9 ?
