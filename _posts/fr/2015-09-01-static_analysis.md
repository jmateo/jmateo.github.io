**Analyse statique Java DIY** par Nicolas Peru (présentation en français)

L'analyse statique permet de détecter des bugs et des problèmes dans votre code sans l'exécuter.
Cette présentation vous promet, en toute simplicité, d'écrire votre propre analyseur pour Java basé sur SonarQube.
Pour y arriver il faudra comprendre les défis d'analyse du langage, du parsing à l'exécution symbolique, pour pouvoir coder des vérifications spécifiques à vos projets !

**Améliorer la qualité de vos tests avec Mutation Testing** par Nicolas Frankel et Evgeny Mandrikov (présentation en anglais)
Les tests unitaires assurent que votre code de production est pertinent. Mais qu'est-ce qui assure que votre code de test est pertinent ? Venez découvrir le mutation testing et assurez-vous de ne jamais oublier une assertion.
Dans le domaine des tests, la métrique de couverture de code est celle dont on parle le plus souvent. Cependant, cela ne signifie pas que le test a été utile ou même qu'une assertion a été codée. Le mutation testing est une stratégie pour s'assurer que le code de test est pertinent.
Dans cette présentation, nous expliquerons comment la couverture de code est calculée et quels sont ses défauts inhérents. Ensuite, nous décrirons comment fonctionne le mutation testing et comment il aide à identifier le code testé mais qui laisse de côté les cas limites. Nous ferons également une démonstration de PIT, un framework Java de qualité production qui permet le mutation testing sur une base de code simple. Si le temps le permet, une démonstration montrera également comment PIT peut être intégré avec SonarQube.
